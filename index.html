We use github-pages for the server. Its a simple setup linking a github repo to a static page. The only downside is that the page is static/no serverside processing is enabled. The bandwidth and disk quota is not strictly specified, but for smaller sites (which the service is intended) the ease of use and stability is fine.

The diagram can give an overview of the tendencies while yielding details interactivle when hovering the cursor over a label on the circel. The downside is that the circular layout of the labels have little meaning and the connections can become too cluttered for any meaning to be seen.
